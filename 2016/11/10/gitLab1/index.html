对gitlab的初步理解

一、安装git，个人觉得windows安装客户端，Linux用yum安装好就可以使用了。
二、按照github/gitlab官网上给出的步骤先创建密钥，将密钥填写至该填写的位置。
三、将本地代码提交至远端服务器
1、本地配置
git config –global user.name ‘caibaoying’
git config –global user.email ‘caibaoying@163.com’ #全局联系方式，可选
2、新建Git项目并提交到Github/gitlab
touch README.md
git init #初始化一个本地库
git add README.md #添加文件到本地仓库
git rm README.md #本地倒库内删除
git commit -m “first commit” #提交到本地库并备注，此时变更仍在本地。
git commit -a ##自动更新变化的文件，a可以理解为auto
git remote add origin git@github.com:xxx/xxx.git #增加一个远程服务器的别名。xxx/xxx.git就是用户名/项目名.git
git remote rm xxx ##删除远程版本库的别名
git push -u origin master #将本地文件提交到Github的remoname版本库中。此时才更新了本地变更到github服务上。

遇到的错误：
我是先touch了一个README.me文件后，提交成功后，我又想把自己的项目再提交上去，结果就出现了问题；项目无法提交成功，我以为是命令有错误，检查之后发现没问题，最后发现问题出现在README.me和项目不在同一个文件夹下，导致出现了问题
解决办法：
gitlab在远端服务器上以Project为单位，github以仓库为单位，所以提交的文件要在相同的文件夹下，注意此问题。
